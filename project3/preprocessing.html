Ways to implement preprocessing

Websites can implement preprocessing in two ways. The website can run scripts in one of two places which are the client side or the server side. Most web coding languages are designed to run on either the server side or the client side. This largely defines how they work. Here are some examples.

Client Side Preprocessing

Client side development is done mostly in JavaScript. JavaScript is ran in addition to basic HTML and CSS code. The reason JavaScript is consider a client side preprocessor is because it runs scripts on the computer after the webpage has loaded.

Server Side Preprocessing

A server side preprocessor runs its scripts before the HTML is loaded.There are various server side languages available. PHP and Ruby are the most popular server side preprocessors but there are many others. The server side preprocessors run their scripts on the server which hosts the website and sends down the HTML code.
